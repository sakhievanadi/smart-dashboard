# SmartHome UI Kit & Dashboard

An interactive, high-performance control panel for Smart Home systems. This project is a pixel-perfect implementation of a modern Figma design, focused on smooth micro-interactions.

## Project Links
- **[Click Here](http://smartdashboardshen.io/)**
- **[Live Demo](https://github.com/sakhievanadi)**
- **[Original Design (Figma)](https://www.figma.com/design/Cdg0yeLxAoGvlQnS0pUkL6/Smart-home-dashboard-free-light--Community-?t=DJM8p6X7gOQtjQfA-0)**

## I paid special attention to the UI/UX details to make the dashboard feel "alive":
- **Smooth "Drop" Navigation:** a custom-built sidebar indicator that physically slides between items using advanced CSS transitions.
- **Dynamic Icon State Management:** smart color-shifting for icons using CSS filters and opacity transitions.
- **Responsive Layout:** built with a "flex-first" approach for adaptability across different screen resolutions.

## Tech Stack
- **HTML5**
- **CSS3**
- **JavaScript (ES6+)**

## 🗺️ Project Roadmap

This dashboard is a living ecosystem. The development is divided into strategic phases to ensure scalable architecture and clean code.

### Phase 1: Core Interactivity & UI (In Progress)
- [x] **Dynamic Sidebar:** Data-driven navigation with smart indicator positioning.
- [x] **Smart Greeting System:** Time-aware greetings (Morning/Afternoon/Evening) based on local system time.
- [ ] **Interactive Device Cards:**
  * [ ] State management for `ON/OFF` toggles.
  * [ ] Dynamic CSS filtering for active/inactive device states.
- [ ] **Climate Control Widget:**
  * [ ] Circular temperature slider implementation.
  * [ ] Real-time value synchronization with internal state.

### Phase 2: External Integrations
- [ ] **Live Weather Engine:**
  * [ ] Connection to **OpenWeather API**.
  * [ ] Automatic icon switching based on weather condition codes (Almaty local data).
- [ ] **User Authentication & Management:**
  * [ ] Registration form for new "Home Members".
  * [ ] Role-based access badges (Admin, Full Access, Guest).

### Phase 3: Data & Analytics
- [ ] **Energy Consumption Visuals:**
  * [ ] Implementation of **Chart.js** or **D3.js** for monthly power usage tracking.
  * [ ] Custom gradient-filled line charts to match the UI/UX design.
- [ ] **Environmental Monitoring:**
  * [ ] Real-time Air Quality Index (AQI) tracking.

### Phase 4: Optimization & Refinement
- [ ] **Sleek Dark Mode:** A high-contrast night-theme variant using CSS Variables.
- [ ] **Advanced Animations:** Smooth `Cubic-Bezier` transitions for all state changes.
- [ ] **Performance Audit:** Image optimization and code minification for faster loading.
